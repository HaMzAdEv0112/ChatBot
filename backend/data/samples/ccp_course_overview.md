# Complex Computing Problem (CCP) — Course Overview

## What is CCP?

Complex Computing Problem (CCP) is a university module that challenges students to design, implement, and evaluate a software solution to a real-world computing problem. The module emphasizes:

- Problem analysis and requirements gathering
- System design and architecture
- Implementation using appropriate technologies
- Testing, evaluation, and documentation
- Professional communication and presentation skills

## Learning Outcomes

By the end of the CCP module, students should be able to:

1. **Analyse** a complex computing problem and define clear requirements
2. **Design** a scalable and maintainable system architecture
3. **Implement** a working solution using modern development practices
4. **Evaluate** the solution against defined success criteria
5. **Document** the project thoroughly for technical and non-technical audiences
6. **Present** findings and demonstrate the working system

## Assessment Components

| Component | Weight | Description |
|-----------|--------|-------------|

## Recommended Technologies

Students may choose technologies appropriate to their problem domain. Common choices include:

- **Frontend:** React, Vue.js, or Angular
- **Backend:** Python (FastAPI/Django), Node.js (Express), or Java (Spring)
- **Database:** PostgreSQL, MongoDB, or SQLite
- **AI/ML:** Python with scikit-learn, TensorFlow, or RAG pipelines
- **DevOps:** Docker, GitHub Actions, cloud deployment (AWS/Azure/Vercel)

## RAG Chatbot Project Idea

A Retrieval-Augmented Generation (RAG) chatbot is an excellent CCP project because it combines:

- **Natural Language Processing:** Understanding user queries
- **Information Retrieval:** Searching a knowledge base for relevant context
- **Large Language Models:** Generating human-like responses
- **Full-Stack Development:** React frontend + API backend
- **Real-World Application:** University helpdesk, course FAQ, or study assistant

### RAG Pipeline Steps

1. **Document Ingestion:** Load PDFs, text files, or markdown into the system
2. **Chunking:** Split documents into smaller passages for retrieval
3. **Embedding:** Convert text chunks into vector representations
4. **Vector Store:** Store embeddings in a database (e.g., ChromaDB)
5. **Retrieval:** Find the most relevant chunks for a user query
6. **Generation:** Pass retrieved context + query to an LLM for the final answer
